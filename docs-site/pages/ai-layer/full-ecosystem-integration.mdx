
# Full AI Ecosystem Integration

## Architecture Overview

MtaaDAO's intelligence layer consists of three tiers working in concert:

1. **AI Layer** - User-facing intelligence (NURU, KWETU, MORIO)
2. **Core Agents** - System-level operations (ANALYZER, DEFENDER, SYNCHRONIZER)
3. **Elder Agents** - Specialized advanced capabilities (7 Elders in development)

This document details how all components interact, make decisions, and communicate with users.

## Interaction Patterns

### Pattern 1: User Query ‚Üí AI Response

**Example:** User asks "What's our DAO balance?"

```
1. User types in MORIO chat
   ‚îî‚Üí MORIO receives query

2. MORIO ‚Üí NURU: "Understand this query"
   ‚îî‚Üí NURU classifies intent: "treasury_balance"
   ‚îî‚Üí NURU extracts entities: { daoId: "abc123" }

3. NURU ‚Üí KWETU: "Execute getTreasuryBalance(abc123)"
   ‚îî‚Üí KWETU queries database
   ‚îî‚Üí KWETU returns: { balance: "15000 cUSD" }

4. KWETU ‚Üí NURU: Raw data
   ‚îî‚Üí NURU formats insight: "Healthy balance, +5% from last week"

5. NURU ‚Üí MORIO: Formatted response
   ‚îî‚Üí MORIO generates natural language:
       "Your DAO has 15,000 cUSD! That's actually up 5% 
        from last week. Looking good! üéâ"

6. MORIO ‚Üí User: Friendly response
```

**Decision Points:**
- NURU decides what data to fetch
- KWETU decides if user has permission
- MORIO decides tone and language

**Notifications:**
- No notifications (informational query)

### Access to Decision Logs

**Super User:**
```sql

ORDER BY timestamp DESC;
-- Sees ALL decisions across all agents
```

**DAO Admin:**
```sql
SELECT * FROM agent_decision_logs
WHERE affectedEntities CONTAINS 'dao-abc123'
AND timestamp > NOW() - INTERVAL '30 days';
-- Sees decisions affecting their DAO
```

**Regular User:**
```sql
SELECT * FROM agent_decision_logs
WHERE affectedEntities CONTAINS 'user-123'
AND decision->>'action' IN ('QUARANTINE', 'ALERT')
AND timestamp > NOW() - INTERVAL '7 days';
```

## Notification Framework

### Notification Levels

**Level 1: Informational**
- User queries answered
- Routine operations completed
- No action required

**Recipients:** User who initiated action

**Delivery:** MORIO chat response only

---

**Level 2: Low Priority**
- Proposal milestones
- Vault balance changes
- Community updates

**Recipients:** Relevant users (opt-in)

**Delivery:** 
- In-app notification bell
- Email digest (daily)
- MORIO chat (if active)

---

**Level 3: Medium Priority**
- Voting deadlines
- Treasury warnings
- Governance anomalies

**Recipients:** DAO admins + affected users

**Delivery:**
- In-app notification (immediate)
- Email (immediate)
- MORIO chat message
- Dashboard alert (admins)

---

**Level 4: High Priority**
- Security threats
- Fraud detected
- State drift critical

**Recipients:** Super User + DAO admins + affected users

**Delivery:**
- Dashboard alert (immediate, cannot dismiss)
- Email (immediate)
- MORIO chat (immediate, persistent)
- SMS (if configured)
- Webhook to external systems

---

**Level 5: Critical**
- System failures
- Catastrophic threats
- Data integrity issues

**Recipients:** Super User + engineering team + stakeholders

**Delivery:**
- All methods above +
- PagerDuty/incident management
- Phone call escalation
- Status page update

### Elder Decision Authority

**Decision Hierarchy:**
1. LUMEN can override any decision on ethical grounds

- Can override any agent/elder decision
- Must provide reason (logged)
