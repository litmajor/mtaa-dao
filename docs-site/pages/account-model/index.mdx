# Account Model

MtaaDAO's account model is designed for **accessibility**, **security**, and **flexibility**. We support multiple user profiles and permission structures to accommodate different organizational needs.

## Key Features

- **Sub-Profiles**: Create specialized accounts for different roles within your DAO
- **Role-Based Access Control**: Define granular permissions for each profile
- **Multi-Signature Support**: Require multiple approvals for critical actions
- **Social Recovery**: Recover accounts using trusted guardians
- **Mobile-First**: Optimized for mobile money integration

## Account Types

### 1. Individual Account
Personal account for DAO members. Holds voting rights, treasury access, and profile data.

### 2. DAO Treasury Account
The primary account that holds community funds. Controlled by DAO governance rules.

### 3. Admin Account
Administrative account with elevated permissions for DAO management and configuration.

### 4. Sub-Account / Service Account
Specialized accounts for specific functions (e.g., grant distribution, loan management).

## Permission Tiers

```
Level 1: Member
├─ Vote on proposals
├─ View treasury
└─ Access community chat

Level 2: Core Team
├─ Create proposals
├─ Manage members
├─ Access analytics
└─ Submit expense reports

Level 3: Admin
├─ Execute approved proposals
├─ Manage treasury rules
├─ Configure DAO settings
├─ Moderate community
└─ Issue credentials
```
