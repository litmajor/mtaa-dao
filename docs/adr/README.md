ADRs — Architecture Decision Records

This folder contains short Architecture Decision Records (ADRs). Use the ADR pattern to capture why important technical decisions were made.

- Keep each ADR short (~200 words).
- Use the template in `template.md`.
- Organize ADRs by area: `client/`, `server/`, `contracts/`, `agents/`.

When you make a change, create a new file with an incremented numeric prefix, e.g. `0002-why-we-chose-x.md`.
