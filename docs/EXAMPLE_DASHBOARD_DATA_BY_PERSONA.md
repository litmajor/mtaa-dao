## Key Insights from This Example

### Alex's Multi-Mode Lifestyle

**Primary Purpose:** Community governance (primary persona: okedi)
- Creates DAOs
- Proposes initiatives
- Votes on governance

**Secondary Purpose:** Execute trades actively
- Maintains positions with leverage
- Watches market alerts
- Trades 5-10 times per month
- Can switch to Trader mode when focused on this

**Tertiary Purpose:** Passive wealth accumulation
- Invests in DAOs
- Yield farms
- Build long-term wealth
- Can switch to Investor mode for this focus

**Why Mode Switching Helps:**
- Community mode: Shows DAOs first, governance primary
- Trader mode: Shows positions first, trading primary
- Investor mode: Shows wealth growth, passive income primary

**All data is always there**, just different UI organization.

### Dashboard Data Volume by Mode

| Data Type | Community | Trader | Investor |
|-----------|-----------|--------|----------|
| DAOs | Prominent | Secondary | Primary |
| Governance | Prominent | Secondary | Secondary |
| Trading | Hidden | Prominent | Hidden |
| Yield | Secondary | Prominent | Prominent |
| Passive Income | Secondary | Secondary | Prominent |
| Portfolio Value | Hidden | Secondary | Prominent |
| Win Rate Stats | Hidden | Prominent | Hidden |
| Market Alerts | Hidden | Prominent | Hidden |

## This is the Right Mental Model

User doesn't become a "Community person" locked out of trading.  
User becomes a "Community-focused person" who can still trade.

All features. Organized by focus. Switchable anytime.
