Audit integrations with external services.

Check:
- API reliability
- timeout handling
- retries/backoff
- fallback logic

Output:
- risks
- improved integration strategy
- monitoring recommendations
- security considerations
- A md file that maps out the integration points, potential failure modes, and mitigation strategies.
- A listt of all third-party services used, along with their data shared, security implications, and monitoring status.
- A report on the current state of external integrations, highlighting any areas of concern and actionable recommendations for improvement.
- A set of best practices for managing external integrations, including guidelines for selecting services, handling failures, and ensuring security.
