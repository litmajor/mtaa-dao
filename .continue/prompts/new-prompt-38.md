Audit transactional behavior.

Check:
- partial updates
- race conditions
- missing rollback
- isolation level issues
- improper error handling
- concurrency issues
- transactional boundaries
- idempotency issues
- deadlocks
- long-running transactions
- distributed transaction issues
- improper use of transactions
- inconsistent state handling
- improper locking
- retry logic issues
- improper use of savepoints

Output:
- risks
- corrected transaction handling
